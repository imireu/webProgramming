Chapter 1: 웹기초 / 웹프론트엔드

웹의 기본 구조: 클라이언트-서버 모델

- 클라이언트(Client): 서비스를 요청하는 주체. 대표적으로 웹 브라우저
- 서버(Server): 요청을 처리하고 응답하는 컴퓨터. 웹서버, 앱서버 등

클라이언트 구성 요소

- 웹 브라우저: 사용자가 URL을 입력하면 서버에 요청을 보내고 HTML 결과를 받아 화면에 렌더링함
- 예: Chrome, Firefox, Edge 등

서버 구성 요소

- 웹 서버(Web Server): 클라이언트의 요청을 받아 정적 자원(HTML, CSS, JS)을 응답함
- 애플리케이션 서버(App Server): 동적 웹페이지 생성, DB 연동 등 백엔드 로직 처리

웹의 3요소

  • HTTP: 클라이언트와 서버 간 통신 규약
  • URL: 웹에서 자원의 위치
  • HTML: 웹페이지 구조를 정의하는 마크업 언어

URL 상세 설명

1. URI / URL / URN

  • URI (Uniform Resource Identifier): 자원을 식별하는 전체 개념. URL과 URN 포함
  • URL (Locator): 자원의 위치와 접근 방법을 명시 (예: https://example.com)
  • URN (Name): 자원의 고유한 이름을 식별. 위치와는 무관 (예: urn:isbn:978-3-16-148410-0)

2. URL 구성 요소

예시: https://www.example.com:443/path/index.html?search=query#section

  • 스키마 (Scheme): https — 자원에 접근할 프로토콜 (http, https, ftp 등)
  • 호스트 (Host): www.example.com — 자원이 위치한 서버 주소
  • 포트 (Port): 443 — 서버와 통신할 포트 번호 (기본 포트는 생략 가능)
  • 경로 (Path): /path/index.html — 서버 내부 자원의 경로
  • 쿼리 문자열 (Query String): ?search=query — 클라이언트가 서버에 전달하는 파라미터
  • 프래그먼트 (Fragment): #section — 페이지 내 특정 위치로 이동

DNS와 요청-응답 흐름

  1. 브라우저 주소창에 URL 입력
  2. DNS 서버에 요청 → 도메인을 IP 주소로 변환
  3. 변환된 IP로 웹 서버에 요청 전송
  4. 서버가 요청을 처리하고 응답(HTML, CSS, JS 등)을 반환
  5. 브라우저가 응답 받은 HTML을 해석해 페이지를 화면에 렌더링
