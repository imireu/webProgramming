Chapter 5: 반복문과 함수

1. 반복문

for, while, do-while

  • for문: 반복 횟수가 명확할 때 사용. 반복 변수를 초기화하고 조건을 검사하며, 반복 후 증감함.
  • while문: 반복 조건이 참일 동안 계속 실행. 반복 횟수가 불확실할 때 유용.
  • do-while문: 코드를 최소 1회 실행한 후 조건을 검사.
for (let i = 0; i < 5; i++) {
  console.log(i);
}

let i = 0;
while (i < 5) {
  console.log(i);
  i++;
}

i = 0;
do {
  console.log(i);
  i++;
} while (i < 5);

논리 연산자 (AND / OR / NOT)

  • && (AND): 좌우 조건이 모두 참일 때 전체가 true
  • || (OR): 둘 중 하나만 참이어도 true
  • ! (NOT): 값의 반대를 반환
console.log(true && false); // false
console.log(true || false); // true
console.log(!true); // false

두 수가 모두 짝수인지 확인

prompt 창으로 입력받은 두 수가 모두 짝수인지 확인

const num1 = parseInt(prompt("첫 번째 수를 입력하세요"));
const num2 = parseInt(prompt("두 번째 수를 입력하세요"));

if (num1 % 2 === 0 && num2 % 2 === 0) {
  alert("두 수 모두 짝수입니다.");
} else {
  alert("두 수 모두 짝수가 아닙니다.");
}

단축 평가

논리 연산자를 활용하여 조건문을 간단하게 작성할 수 있음

// OR: 기본값 설정
let username = inputName || "익명 사용자";

// AND: 조건이 true일 때만 함수 실행
isLoggedIn && showDashboard();

배열 반복문

배열의 각 요소를 순회하며 작업을 수행할 수 있음

const fruits = ["사과", "바나나", "오렌지"];

// 전통적 for문
for (let i = 0; i < fruits.length; i++) {
  console.log(fruits[i]);
}

// forEach 메서드
fruits.forEach(function(fruit) {
  console.log(fruit);
});

forEach의 특징

  • 콜백 함수를 통해 배열 요소에 대해 반복 작업 수행
  • return으로 루프를 빠져나갈 수 없음

화살표 함수

function 키워드 없이 간단한 문법으로 함수를 정의할 수 있음

fruits.forEach(fruit => console.log(fruit));

for...in / for...of

for...in: 객체의 속성(key)을 순회할 때 사용
for...of: 배열, 문자열 등 이터러블 객체의 값을 순회할 때 사용 

const user = { name: "홍길동", age: 25 };
for (let key in user) {
  console.log(key + ": " + user[key]);
}

const colors = ["red", "green", "blue"];
for (let color of colors) {
  console.log(color);
}

중첩 반복문 - 구구단

for (let i = 2; i <= 9; i++) {
  for (let j = 1; j <= 9; j++) {
    console.log(`${i} x ${j} = ${i * j}`);
  }
}

별 찍기 (do-while)

사용자에게 입력받은 수만큼 별 출력

let stars = parseInt(prompt("별 개수를 입력하세요"));
let count = 0;
do {
  console.log("*");
  count++;
} while (count < stars);

break / continue

  • break: 반복문을 즉시 종료
  • continue: 이후 코드 생략, 다음 반복으로 진행
for (let i = 1; i <= 5; i++) {
  if (i === 3) continue; // 3 생략
  if (i === 5) break;    // 5에서 반복 종료
  console.log(i);
}

2. 함수와 스코프

함수를 사용하는 이유

중복 제거, 코드 재사용, 유지보수 용이, 논리적 구조화 등 장점이 있음

함수 선언과 실행

function sayHello() {
  console.log("안녕하세요!");
}

sayHello();

매개변수와 인수

함수를 정의할 때 사용하는 변수: 매개변수
함수를 호출할 때 전달하는 값: 인수

function greet(name) {
  console.log(`${name}님, 환영합니다!`);
}

greet("지민");

return 문

함수 실행 결과를 반환하고, 함수 실행을 종료

function add(a, b) {
  return a + b;
}

const result = add(3, 4);
console.log(result);

기본 매개변수

매개변수에 기본값을 지정해 인수가 생략된 경우에도 오류 없이 처리

function greet(name = "손님") {
  console.log(`${name}님, 안녕하세요!`);
}

greet();

디버깅 - 브레이크포인트

개발자 도구(F12)에서 특정 줄에 중단점을 설정해 함수 내부 상태를 추적 가능

스코프 (Scope)

  • 함수 레벨 스코프: var는 함수 내부에서만 지역 변수로 취급
  • 블록 레벨 스코프: let, const는 중괄호 내에서만 유효

var 예약어 주의

var는 중복 선언 가능하고, 함수 외에서도 전역으로 선언되는 문제가 있어 사용 주의

호이스팅

변수 및 함수 선언이 코드 실행 전 위로 끌어올려지는 현상

console.log(x); // undefined 출력됨
var x = 5;

변수 사용 시 권장사항

  • let, const 사용 권장
  • 불필요한 전역 변수 사용 지양
  • 변경되지 않는 값에는 const 사용

3. 함수 표현식

함수 정의 방식

  • 함수 선언식: function foo() {...} (호이스팅 됨)
  • 함수 표현식: const foo = function() {...} (호이스팅 안 됨)

즉시 실행 함수 (IIFE)

정의와 동시에 한 번만 실행되는 함수

(function() {
  console.log("즉시 실행 함수입니다.");
})();

화살표 함수

간결한 문법과 this 바인딩 없음

const add = (a, b) => a + b;
console.log(add(2, 3));

콜백 함수

다른 함수에 인수로 전달되어 특정 조건 또는 시점에 실행되는 함수

function processUserInput(callback) {
  const name = prompt("이름을 입력하세요");
  callback(name);
}

processUserInput(name => {
  alert("안녕하세요, " + name + "님!");
});

전개 구문 (Spread Syntax)

배열이나 객체의 값을 펼쳐서 전달할 수 있음

const nums = [1, 2, 3];
console.log(...nums); // 1 2 3

구조 분해 할당

배열이나 객체의 값을 쉽게 변수에 할당

const [a, b] = [1, 2];
const { name, age } = { name: "민수", age: 30 };

타이머 함수

  • setTimeout: 일정 시간 후 한 번 실행
  • setInterval: 일정 시간 간격으로 반복 실행
  • clearInterval: 반복 종료
const intervalId = setInterval(() => {
  console.log("반복 실행 중");
}, 1000);

setTimeout(() => {
  clearInterval(intervalId);
  console.log("반복 종료");
}, 5000);
