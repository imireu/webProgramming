Chapter 2: 웹프론트엔드 (2)

자바스크립트 역할

  • 동적 처리
  • 상호작용 처리
  • 접근성 개선
  • 사용자 경험(UX) 개선
  • 애플리케이션 개발 (SPA, PWA 등)
  • 브라우저 내 제어 (폼 검증, 이벤트 핸들링 등)

자바스크립트 작성법

  • 인라인 방식: HTML 태그 내 직접 삽입
  • 인터널 방식: <script> 태그 내부에 작성
  • 익스터널 방식: 외부 JS 파일로 분리하여 <script src="...">
  • 필요할 때 DOM을 통해 코드 생성 및 삽입

HTML & 웹 버전

  • HTML 1.0 ~ 4.01: 문서 구조 중심, 점차 확장
  • HTML5: 시맨틱 태그, 멀티미디어 지원, API 추가 등 현대 웹에 맞게 개편

HTML5에 추가된 주요 자바스크립트 API

  • Geolocation API
  • Drag & Drop
  • Web Storage (localStorage, sessionStorage)
  • Application Cache (현재는 deprecated)
  • Web Workers
  • Server-Sent Events

TypeScript

정적 타입을 가진 자바스크립트의 상위 버전

  • 장점: 컴파일 단계에서 오류 확인 가능, 코드 완성도 및 유지보수 향상
  • 단점: 학습 곡선, 빌드 필요

프론트엔드 프레임워크 및 라이브러리

  • React: 컴포넌트 기반 UI 라이브러리
  • Vue: 쉽고 빠르게 UI 구축 가능한 프레임워크
  • Angular: 대규모 앱 구축에 적합한 프레임워크
  • jQuery: DOM 제어 편의성 제공
  • Next.js, Vite, Astro 등 번들링/라우팅 도구

ECMAScript (ES)

자바스크립트의 표준 사양. 최신 버전은 ES2023까지 발표됨.

HTML Tags

참조 사이트: W3C, Mozilla MDN

  • Basic Tags: html, head, body, div, span, h1~h6, p 등
  • Forms & Inputs: form, input, select, textarea, button
  • Frames: iframe 등
  • Media: img, audio, video
  • Links: a, nav
  • Lists: ul, ol, li, dir, dl, dt, dd
  • Tables: table, thead, tbody, tr, th, td
  • Style & Semantics: style, b, strong, em, mark
  • Meta Info: title, meta, link
  • Metadata Content: base, meta, noscript, script, style, title
  • Programming: script, noscript, canvas

CSS 개요

  • HTML은 구조, CSS는 표현, JS는 동작
  • CSS 기능: 색상, 크기, 레이아웃, 애니메이션 등
  • 작성 위치: 인라인, 인터널, 익스터널
  • 적용 순서: 인라인 > 인터널 > 익스터널 (Cascading)

CSS 셀렉터 예시

  • 전체 선택자: *
  • 태그 선택자: p
  • 클래스 선택자: .class
  • 아이디 선택자: #id
  • 자식 선택자: div > p

CSS 선언 형식


선택자 {
  속성: 값;
  속성: 값;
}

컬러 지정 방식

  • 이름: red, blue 등
  • Hex: #ff0000
  • RGB: rgb(255, 0, 0)
  • RGBA: rgba(255, 0, 0, 0.5)
  • HSL: hsl(0, 100%, 50%)
