Chapter 4: JavaScript 기초 문법

1. 자바스크립트 - 변수

alert, confirm, prompt 함수

  • alert("메시지"): 경고창 표시
  • confirm("질문"): 확인/취소 버튼이 있는 창
  • prompt("입력 요청"): 사용자로부터 입력값을 받는 창
alert("경고입니다!");
confirm("정말 삭제하시겠습니까?");
prompt("이름을 입력하세요");

콘솔창의 undefined 의미

값을 명시적으로 반환하지 않는 표현식의 결과. 예를 들어, 변수 선언만 했을 경우 undefined가 출력됨

let a;
console.log(a); // undefined

console.log(), document.write()

  • console.log(): 콘솔에 메시지를 출력
  • document.write(): 문서에 직접 텍스트 출력 (실무에선 잘 사용되지 않음)

변수 정의 및 규칙

  • 문자, 숫자, $, _ 사용 가능 (단 숫자로 시작 불가)
  • 예약어 사용 불가, 대소문자 구분
  • camelCase로 작성 권장

상수와 변수의 필요성

변수는 변경 가능한 값을 저장할 때, 상수는 변경되면 안 되는 값을 저장할 때 사용

var / let / const 차이

  • var: 함수 스코프, 재선언/재할당 가능 (ES6 이전 방식, 권장하지 않음)
  • let: 블록 스코프, 재할당 가능
  • const: 블록 스코프, 재선언/재할당 불가 (상수 선언에 사용)
let name = "홍길동";
name = "임꺽정"; // 가능

const age = 20;
age = 21; // 오류

2. 자바스크립트 - 자료형

자료형 정의

변수에 저장할 수 있는 값의 타입. 자바스크립트는 동적 타이핑 언어로, 변수 선언 시 타입을 지정하지 않고 실행 도중 타입이 변할 수 있음

기본 자료형 (Primitive Types)

  • String: 문자열 (예: "hello")
  • Number: 숫자 (정수, 실수 포함)
  • Boolean: true / false
  • undefined: 값이 없음 (변수 선언만 하고 초기화하지 않은 상태)
  • null: 의도적으로 비워둔 상태
  • Symbol: 고유한 식별자

typeof() 함수

typeof "hello" // "string"
typeof 42 // "number"
typeof undefined // "undefined"
typeof null // "object" (자바스크립트 고질적 버그)

백슬래시(\)와 특수기호

\n : 줄바꿈
\t : 탭
\" : 큰따옴표 출력
\' : 작은따옴표 출력

템플릿 리터럴

문자열과 변수를 쉽게 조합하기 위해 도입. 백틱(`) 사용, ${} 구문으로 변수 삽입

let name = "지민";
console.log(`안녕하세요 ${name}님`);

null과 undefined

  • null: 의도적으로 변수에 비어 있음을 명시
  • undefined: 선언만 되었고 값이 할당되지 않은 상태

배열과 인덱스

let fruits = ["사과", "바나나", "포도"];
console.log(fruits[1]); // 바나나

심볼(Symbol)

고유한 값으로 객체 속성의 키 등에서 충돌을 방지하기 위해 사용

const sym = Symbol("id");

형변환

  • 자동 형변환: 연산이나 조건문에서 자동으로 타입이 변함
  • 명시적 형변환: String(), Number(), Boolean() 함수 사용
let str = "5";
let num = Number(str); // 5

프롬프트 입력 시 자료형

모든 입력은 기본적으로 문자열(String) 형태로 저장되며, 숫자로 사용하려면 Number()를 이용해 형변환해야 함

3. 연산자와 제어문

연산자 정의 및 종류

  • 산술: +, -, *, /, %
  • 비교: ==, ===, !=, !==, >, <,>=, <=< /li>
  • 논리: &&, ||, !
  • 대입: =, +=, -=, *= 등

삼항 연산자

조건에 따라 값을 간단하게 결정할 때 사용

let age = 18;
let result = (age >= 18) ? "성인" : "미성년자";
console.log(result);

연산자 우선순위

괄호() > 단항연산자 > 산술 > 비교 > 논리 > 대입

조건문 구조

if (조건) {
  // 실행문
} else if (조건2) {
  // 다른 조건
} else {
  // 나머지 경우
}

실습: 3의 배수 판별

let input = prompt("숫자를 입력하세요");

if (input === null) {
  alert("입력이 취소되었습니다.");
} else if (input.trim() === "") {
  alert("값을 입력하지 않았습니다.");
} else {
  let number = Number(input);
  if (!isNaN(number)) {
    if (number % 3 === 0) {
      alert("3의 배수입니다.");
    } else {
      alert("3의 배수가 아닙니다.");
    }
  } else {
    alert("유효한 숫자가 아닙니다.");
  }
}

실습: 홀수/짝수 판별

let num = prompt("숫자를 입력하세요");

if (num !== null && num.trim() !== "") {
  let value = Number(num);
  if (!isNaN(value)) {
    if (value % 2 === 0) {
      alert("짝수입니다");
    } else {
      alert("홀수입니다");
    }
  } else {
    alert("숫자를 입력하세요");
  }
} else {
  alert("입력이 없습니다");
}

switch 문

하나의 값을 기준으로 여러 경우(case)를 분기하여 실행할 때 사용. break를 사용하지 않으면 다음 case까지 실행됨에 유의

let fruit = prompt("좋아하는 과일은?");

switch (fruit) {
  case "사과":
    alert("사과를 좋아하시네요!");
    break;
  case "바나나":
    alert("바나나 맛있죠!");
    break;
  case "포도":
    alert("포도는 달콤해요!");
    break;
  default:
    alert("잘 모르는 과일이네요.");
}
